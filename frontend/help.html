Help

Quick Start

Use Alerts to watch store/component health. Click an incident to acknowledge it temporarily. Use Test Sender to generate valid test traffic.

Field Glossary

  • store_id/component: identify the monitored entity.
  • event_type: problem, recovery, disable, or enable.
  • severity: info, warning, or critical.
  • dedup_key: groups repeated reports for the same issue.
  • expected_green_interval_seconds: optional heartbeat timeout window.
  • expires_at: time after which event/ack is no longer valid.

Alert Levels

  • Red: critical issue needing immediate action.
  • Yellow: warning condition requiring awareness.
  • Green: healthy and functional.
  • Purple: heartbeat timeout (missed expected green check-in).
  • White/Clear: monitor disabled; timeout and alerting suppressed.

Acknowledgements

Acknowledging moves an alert to the Acknowledgements page until its expiry time. On expiry it returns to Alerts automatically.

Troubleshooting

  • Connection badge stale/reconnecting indicates websocket interruption.
  • 401 errors indicate API key mismatch.
  • 422 errors indicate invalid field values or expired timestamps.
